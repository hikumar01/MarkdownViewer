# ADR-004: Diagram Renderer — Mermaid.js

**Date:** 2026-05-16
**Status:** Accepted

## Context

MarkdownViewer must render fenced code blocks tagged ` ```mermaid ` as diagrams. The renderer must:

- Run entirely client-side in the WebView — no external server or process
- Support flowcharts, sequence diagrams, Gantt charts, class diagrams, ER diagrams, and pie charts
- Produce SVG output (vector, scalable, selectable text, copy/export friendly)
- Work offline (no CDN or remote API calls)
- Support theme switching (light/dark, plus user-selectable themes)

## Decision

**Use Mermaid.js v11+** for all diagram rendering.

- Initialize with `startOnLoad: false` to control rendering timing
- Render each diagram block using `mermaid.render(id, source)` which returns a Promise
- Sanitize the returned SVG with DOMPurify (SVG profile) before DOM insertion — `securityLevel:'loose'` allows inline SVG output but may include event handlers or `<script>` nodes in malicious diagram source; DOMPurify strips them
- Replace the source `<pre class="mermaid-source">` block with a `<figure class="mermaid-diagram">` containing the sanitized SVG
- Use Mermaid's built-in theme system (`default`, `dark`, `forest`, `neutral`, `base`)

## Rationale

### Pure JavaScript, runs in the WebView

Mermaid.js is a pure TypeScript/JavaScript library with no native code dependencies. It runs directly in the browser environment provided by the Tauri WebView — no separate process, no IPC, no native bindings required.

PlantUML requires a Java runtime or a remote server. Graphviz is a C library that needs a native build or WASM compilation. Both add significant operational complexity for what is a passive viewer app.

### Widest format coverage and adoption

Mermaid covers all diagram types required by the product with a single library. It is the de-facto standard for diagrams in markdown — GitHub, GitLab, Notion, Obsidian, and VS Code all use it. Users authoring `.md` files with ` ```mermaid ` blocks expect MarkdownViewer to render them exactly as those platforms do.

### Async API (v10+)

Mermaid v10+ replaced the synchronous `mermaid.render()` callback with a Promise-based API:

```js
const { svg } = await mermaid.render(id, source)
```

This is non-blocking and integrates cleanly with the async rendering pipeline. Earlier synchronous versions caused layout jank on pages with many diagrams.

### SVG output

SVG output is:
- Scalable without quality loss at any zoom level
- Copyable and exportable (P4 Feature 2 / P7 Feature 3)
- Inspectable for the Diagram Inspector feature (P7 Feature 9)
- Indexable by screen readers when given `role="img"` and a `<title>` (see OP-16)

Raster output (PNG canvas) would lose all of these properties.

### Theme system

Mermaid has a first-class theme API. Switching themes calls `mermaid.initialize({ theme })` and re-renders affected diagrams. This powers P5 Feature 9 (Mermaid Theme Switching) without any custom theming code.

## Consequences

**Committed to:**
- Mermaid.js bundled in the app — not loaded from a CDN
- All diagram types supported by Mermaid v11 are in scope; diagram types not in Mermaid are out of scope for v1
- Mermaid renders asynchronously — the page must handle a "rendering" → "rendered" state transition per diagram
- Invalid diagram source shows Mermaid's built-in error output (not a crash)
- SVG `id` values generated by Mermaid must not collide with heading anchor `id` values — prefix Mermaid IDs with `mermaid-`

**Ruled out:**
- PlantUML (requires Java or remote server)
- Graphviz (C library, WASM path untested at app scale)
- D2 (emerging tool; not yet widely used in `.md` files authors write)

## Alternatives Considered

| Renderer | Verdict |
|---|---|
| PlantUML | Rejected — requires Java runtime or remote API; violates offline requirement |
| Graphviz | Rejected — C library; WASM version is large and not production-ready for complex diagrams |
| D2 | Rejected — too new; authors do not write ` ```d2 ` blocks yet |
| viz.js (Graphviz WASM) | Rejected — ~3 MB WASM binary; limited diagram type coverage vs Mermaid |
