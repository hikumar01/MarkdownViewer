## 1. Command Palette (Cmd+K)

A fuzzy-search overlay for quickly accessing commands, recent files, and document headings.

**Acceptance Criteria**

- `Cmd+K` opens the command palette as a centered floating overlay
- The input field is focused immediately on open
- `Escape` or clicking outside closes the palette without action
- Three categories of results:

  **Commands** (always available):
  - Open File
  - Toggle Table of Contents
  - Toggle Status Bar
  - Toggle Theme (Light / Dark / System)
  - Open in Editor
  - Clear Recent Files

  **Recent Files** (shown when input is empty or matches filename):
  - Last 10 opened files, shown with shortened path

  **Headings** (shown when input is empty or matches heading text):
  - All headings in the current document with level indicator

- Fuzzy matching: typing filters all three categories simultaneously
- Arrow keys navigate the list; `Enter` executes the selected item
- Each result shows an icon indicating its category (command / file / heading)
- Executing a "Go to Heading" result scrolls to that heading and closes the palette

**Technical Notes**

Implement the palette UI as a renderer-side component (no native OS involvement needed). Use a simple fuzzy filter:

```js
function fuzzyMatch(query, text) {
  let qi = 0
  for (const char of text.toLowerCase()) {
    if (char === query[qi]) qi++
    if (qi === query.length) return true
  }
  return false
}
```

Handle `Cmd+K` as an in-renderer keyboard shortcut — no global shortcut registration needed since the palette only opens when the app window is focused:

```typescript
document.addEventListener('keydown', (e) => {
  if ((e.metaKey || e.ctrlKey) && e.key === 'k') {
    e.preventDefault()
    openCommandPalette()
  }
})
```

If the palette must also be triggerable from the menu bar, wire the menu item to emit a Tauri event:

```rust
// app/src/menu.rs
MenuItem::with_id(app, "cmd-palette", "Command Palette", true, Some("CmdOrCtrl+K"))
```

```typescript
import { listen } from '@tauri-apps/api/event'
await listen('menu-cmd-palette', () => openCommandPalette())
```

**Dependencies:** TOC (P3 Feature 1), recent files (P3 Feature 7), status bar (P3 Features 3–5)

## 2. Custom CSS Override

Users can supply a CSS file that is loaded after the default stylesheet, allowing full visual customization.

**Acceptance Criteria**

- A "Custom CSS" setting in Preferences accepts a file path to a `.css` file
- The custom CSS is injected after `github-markdown.css` — it can override any default style
- Changes to the custom CSS file are reflected on the next document reload (auto-watch optional, not required)
- Invalid CSS (syntax errors) is silently ignored — does not break the default styles
- Custom CSS is scoped to the document content area (`.markdown-body`) — it cannot override the app chrome (status bar, TOC panel, title bar)
- Disabling the setting (clearing the path) removes the custom CSS on the next reload
- A "Open Custom CSS File" button in Preferences opens the file in the default editor

**Technical Notes**

Load the custom CSS via a `<link>` element in the renderer:

```js
function applyCustomCss(filePath) {
  const existing = document.getElementById('custom-css')
  if (existing) existing.remove()
  if (!filePath) return
  const link = document.createElement('link')
  link.id = 'custom-css'
  link.rel = 'stylesheet'
  link.href = `markview://${filePath}?bust=${Date.now()}` // cache bust on reload
  document.head.appendChild(link)
}
```

The custom CSS file is served via the same `markview://` custom protocol used for local images (P0 Feature 8).

**Dependencies:** Local image protocol (P0 Feature 8), typography (P1 Feature 2)

## 3. Font Size Controls

The user can increase or decrease the base font size of the document content.

**Acceptance Criteria**

- `Cmd++` increases font size; `Cmd+-` decreases; `Cmd+0` resets to default
- Also accessible via View menu → "Zoom In / Zoom Out / Reset Zoom"
- Font size range: 12px – 24px, in 2px increments
- Default: 16px
- All text scales proportionally (headings, body, captions, code) — everything is relative to the base font size via `em` or `rem` units
- Setting is persisted and restored on relaunch
- Font size only affects the content area — not the status bar or TOC panel chrome

**Technical Notes**

Set the `font-size` on the `.markdown-body` root element. All child elements should use `em` units so they scale automatically:

```js
function setFontSize(px) {
  document.querySelector('.markdown-body').style.fontSize = `${px}px`
  store.set('fontSize', px)
}
```

Handle zoom shortcuts in the renderer with a `keydown` listener — no global shortcut needed since zoom only applies to the focused window:

```typescript
import { Store } from '@tauri-apps/plugin-store'
const store = await Store.load('settings.json')

document.addEventListener('keydown', (e) => {
  if (!(e.metaKey || e.ctrlKey)) return
  if (e.key === '+' || e.key === '=') { e.preventDefault(); adjustFontSize(2) }
  else if (e.key === '-') { e.preventDefault(); adjustFontSize(-2) }
  else if (e.key === '0') { e.preventDefault(); resetFontSize() }
})

async function adjustFontSize(delta: number) {
  const current = await store.get<number>('fontSize') ?? 16
  setFontSize(Math.min(24, Math.max(12, current + delta)))
}

async function resetFontSize() { setFontSize(16) }
```

`Cmd+0` removes the inline style, falling back to the stylesheet default of 16px. Menu items (Zoom In/Out/Reset) are wired to the same functions via Tauri's menu event system.

**Dependencies:** Typography (P1 Feature 2)

## 4. Math / LaTeX (KaTeX)

Inline and display math expressions written in LaTeX syntax are rendered using KaTeX.

**Syntax**

Inline math:
```markdown
The equation $E = mc^2$ describes mass-energy equivalence.
```

Display (block) math:
```markdown
$$
\int_0^\infty e^{-x^2} dx = \frac{\sqrt{\pi}}{2}
$$
```

**Acceptance Criteria**

- Inline math `$...$` renders as inline KaTeX output within the text flow
- Display math `$$...$$` renders as a centered block expression
- KaTeX render errors (invalid LaTeX): display the raw LaTeX source in a red-bordered error span — no document crash
- KaTeX is rendered client-side — no network requests (KaTeX fonts and CSS bundled)
- Rendered math respects light/dark mode (KaTeX default colors work on both; verify contrast)
- Math inside table cells renders correctly
- `$` used as a currency symbol in non-math contexts (e.g., "costs $5") does not accidentally trigger math mode — dollar signs not followed by a matching closing `$` on the same line are left as plain text

**Technical Notes**

Use `remark-math` + `rehype-katex`:

```js
import remarkMath from 'remark-math'
import rehypeKatex from 'rehype-katex'

processor
  .use(remarkMath)
  .use(rehypeKatex)
```

Bundle KaTeX CSS and fonts locally — do not load from a CDN. Add the KaTeX stylesheet to the renderer's `<head>`:

```html
<link rel="stylesheet" href="./katex/katex.min.css">
```

The `$...$` single-dollar inline delimiter has known ambiguity with currency symbols. `remark-math` uses conservative parsing: a `$` only starts a math span if followed by non-whitespace and closed by a matching `$` with non-whitespace before it. Verify this behavior with test fixtures including `$5.00` and `$5 and $10`.

**Dependencies:** remark pipeline (P0 Feature 1)

## 5. macOS Quick Look Plugin

> **Scope: macOS only.** This feature uses the macOS-exclusive `QLPreviewingController` API and has no cross-platform equivalent in v1. See ADR-007 and OP-21 for the Windows discussion.

A Quick Look generator allows Finder to preview `.md` files by pressing Space — without opening markview.

**Acceptance Criteria**

- Pressing Space on a `.md` file in Finder shows a rendered markdown preview
- Mermaid diagrams render in the preview
- Code blocks are syntax-highlighted
- Preview uses markview's light-mode styling
- Preview is read-only — no interaction is supported (Quick Look standard)
- The plugin works even when the markview app is not running
- The plugin is bundled inside the markview app at `markview.app/Contents/PlugIns/MarkviewQuickLook.appex`
- Plugin respects macOS App Sandbox requirements
- Preview updates if Quick Look is open and the file is re-saved (standard Quick Look behavior)

**Implementation Approach**

The Quick Look extension is a separate Xcode target (`QLPreviewingController`). It requires:

1. A Swift/AppKit extension target in the build system
2. A bundled JavaScript runtime (or `WKWebView`) to run the remark/mermaid pipeline
3. The extension passes the `.md` file content to the JS renderer and returns HTML
4. The HTML is displayed in a `WKWebView` inside the Quick Look panel

Option A — Headless JS runtime: Bundle a lightweight JS runtime (e.g., JavaScriptCore) that runs the remark + Mermaid pipeline. Larger bundle but fully offline.

Option B — WKWebView with bundled HTML: Bundle a self-contained HTML page containing the remark + Mermaid JS bundle. The extension loads the page in a `WKWebView` and sends the file content via `evaluateJavaScript`. Simpler to implement.

**Option B is recommended** for implementation simplicity. The bundled HTML page is the same renderer used by the main app.

**Signing and Distribution**

- The extension must be code-signed with the same Developer ID as the main app
- If distributed via the Mac App Store, the extension must pass sandbox review
- If distributed outside the MAS (direct download), sandbox is optional but recommended

**Dependencies:** Entire P0 + P1 rendering stack must be stable; the plugin bundles a snapshot of the renderer

## 6. Copy Rendered HTML of Selection

The user can copy a selection of rendered markdown as HTML, not as plain text.

**Acceptance Criteria**

- Right-clicking a text selection shows "Copy as HTML" in the context menu (alongside the standard "Copy" option)
- The copied HTML contains the rendered output of the selected content — headings, bold, links, tables — not the raw markdown source
- Copied HTML is valid and self-contained enough to paste into an HTML document (no broken `<img>` relative paths — images are omitted from the copy or converted to `data:` URIs)
- Pasting into a rich-text editor (e.g., Google Docs, Notion) produces formatted content
- If no text is selected, the menu item is disabled

**Technical Notes**

Access the user's selection via the Selection API:

```js
function copySelectionAsHtml() {
  const selection = window.getSelection()
  if (!selection.rangeCount) return

  const range = selection.getRangeAt(0)
  const fragment = range.cloneContents()
  const wrapper = document.createElement('div')
  wrapper.appendChild(fragment)
  const html = wrapper.innerHTML

  // Write both text/html and text/plain to clipboard
  navigator.clipboard.write([
    new ClipboardItem({
      'text/html': new Blob([html], { type: 'text/html' }),
      'text/plain': new Blob([selection.toString()], { type: 'text/plain' })
    })
  ])
}
```

Register the custom context menu item by intercepting the `contextmenu` event in the renderer. In Tauri, context menus are built with `@tauri-apps/api/menu`:

```typescript
import { Menu, MenuItem } from '@tauri-apps/api/menu'

document.addEventListener('contextmenu', async (e) => {
  const selection = window.getSelection()
  if (!selection || selection.isCollapsed) return
  e.preventDefault()

  const menu = await Menu.new({
    items: [
      await MenuItem.new({ text: 'Copy as HTML', action: copySelectionAsHtml }),
      await MenuItem.new({ text: 'Copy', action: () => document.execCommand('copy') }),
    ]
  })
  await menu.popup()
})

**Dependencies:** remark pipeline (P0 Feature 1)

## Open Points

### OP-21 — Quick Look: Windows and Linux Equivalents
Feature 5 is macOS-only (Quick Look via `QLPreviewingController`). No equivalent is specified for other platforms:

- **Windows:** File Explorer's Preview Pane uses the `IPreviewHandler` COM interface. It is a separate Windows-only binary component, complex to implement and sign, and would require a different build pipeline entirely.
- **Linux:** No standard equivalent exists across desktop environments (GNOME Nautilus and KDE Dolphin each have their own preview extension APIs).

Decision needed:
- **Option A:** macOS-exclusive feature — document this explicitly and close the gap for Windows/Linux.
- **Option B:** Windows Preview Pane support added to P7.
- **Option C:** A cross-platform alternative — a standalone CLI (`markview-preview <file>`) that opens a lightweight browser window. Lower fidelity than native Quick Look but works on all platforms.

**Recommended: Option A for v1**, with Option C noted in P7 as a future cross-platform alternative.

### OP-22 — Keyboard Shortcuts Conflict Review
The following shortcuts are assigned across requirements. A conflict audit is needed before the menubar is built (see P0 OP-03):

| Shortcut | Feature | Conflict risk |
|---|---|---|
| `Cmd+O` | Open file | None |
| `Cmd+E` | Open in editor | None |
| `Cmd+F` | In-document search | None |
| `Cmd+K` | Command palette | None |
| `Cmd+Shift+T` | Toggle TOC | None |
| `Cmd+[` | Navigate back | Standard browser/app back — fine |
| `Cmd+]` | Navigate forward | Standard browser/app forward — fine |
| `Cmd++` | Zoom in | None |
| `Cmd+-` | Zoom out | None |
| `Cmd+0` | Reset zoom | None |
| `Cmd+Shift+P` | Presentation mode (P7) | **Conflict** — VS Code Command Palette. Primary users are developers. **Recommended replacement:** `Cmd+Shift+F5` or `Ctrl+Shift+P` (less likely to conflict) |
| `Cmd+Shift+O` | Open folder (P7) | None |

`Cmd+Shift+P` must be reassigned before P7 implementation begins.
